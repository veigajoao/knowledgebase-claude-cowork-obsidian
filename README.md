# LLM Knowledge Base — Cowork Setup

A personal wiki system powered by Claude in Cowork. Drop raw sources in, get a
structured, interlinked wiki out. Query it, lint it, grow it over time.

Inspired by [this workflow](https://karpathy.ai) by Andrej Karpathy.

---

## How it works

```
raw/ → kb-ingest → kb-compile → wiki/
                                  ↓
                               kb-qa (answer questions → output/)
                               kb-lint (health checks, gap finding)
```

Raw sources go into `raw/`. Claude reads them, extracts concepts, and compiles
structured markdown articles into `wiki/`. You query the wiki with natural
language. Claude writes all wiki content — you rarely touch it directly.

## Setup

### 1. Create the folder structure in Obsidian (or anywhere)

```
your-kb-folder/
├── raw/        ← drop source files here
├── wiki/       ← Claude writes here
├── index/      ← master index, ingest logs, topic summaries
├── output/     ← Q&A answers, slides, charts
└── tools/      ← helper scripts Claude writes
```

Make `raw/` read-only so Claude can never accidentally modify source files:

```bash
chmod -R 555 /path/to/your-kb-folder/raw
```

### 2. Create a Cowork Project

- Open Cowork → Projects → New project
- Name it after your knowledge base topic
- Under **Context → On your computer**, link your kb folder
- Paste the contents of `project-instructions.md` into the **Instructions** field
- Add this line at the top of instructions:
  > This project uses four skills: kb-ingest, kb-compile, kb-qa, and kb-lint. Always prefer these over ad-hoc approaches for all knowledge base operations.

### 3. Install the skills

Each folder in `skills/` is a Cowork skill. To install:

1. Zip the skill folder and rename it to `skill-name.skill`
   ```bash
   cd skills && zip -r kb-ingest.skill kb-ingest/
   ```
   Or use the skill-creator packaging script if you have it available.
2. Open Cowork → drag the `.skill` file into the window, or use Settings → Skills → Install

Install all four: `kb-ingest`, `kb-compile`, `kb-qa`, `kb-lint`.

### 4. Obsidian plugins (optional but recommended)

- **Marp** — renders slide decks generated by kb-qa
- **Dataview** — lets you query wiki articles like a database
- **Templater** — applies article templates automatically
- **Obsidian Web Clipper** (browser extension) — clips web articles into `raw/` in one click

## Usage

### Day-to-day workflow

| What you say | Skill triggered | What happens |
|---|---|---|
| "Ingest the new files in raw/" | kb-ingest | Reads sources, writes ingest report to index/ |
| "Compile the wiki" | kb-compile | Turns ingest report into wiki articles, updates index |
| "Ingest new files and compile the wiki" | both | Full pipeline in one go |
| "What do I know about X?" | kb-qa | Reads wiki, writes answer to output/ |
| "Compare X and Y" | kb-qa | Writes a comparison table to output/ |
| "Make slides about X" | kb-qa | Writes a Marp slideshow to output/ |
| "Run a health check" | kb-lint | Finds broken links, gaps, stubs, suggests new articles |

### Output formats

kb-qa automatically picks the right format:
- Factual/analytical question → Markdown `.md`
- Overview or presentation → Marp `.md` slideshow
- Trends, comparisons, data → Python chart → `.png`
- Side-by-side comparison → Markdown table

All outputs saved to `output/YYYY-MM-DD-slug.ext`.

### Companion notes

When dropping a source into `raw/`, you can add context by mentioning it in your
message to Claude:

> "Ingest this paper — I found it while reading about X and I'm curious whether it
> addresses Y."

kb-ingest will automatically write a `source-filename.pdf.note.md` companion file
into `raw/` capturing your context before proceeding. That note travels through the
ingest report and into the wiki article's Sources section, preserving your research
trail alongside the source content.

You can also write the note yourself — just create `source-filename.note.md` next
to the source file before ingesting. kb-ingest will use it as-is.

If you say nothing, no note is generated. It's always optional.

### Ingest log

kb-ingest keeps `index/ingest-log.md` — a record of every file processed, so it
never re-ingests the same file twice. Ingest reports are saved as
`index/ingest-report-YYYY-MM-DD-HHMM.md` (timestamped so multiple runs per day
don't overwrite each other).

## Recreating on a new machine

1. Clone this repo
2. Make sure your kb folder is accessible (restore from backup, sync via iCloud/Obsidian Sync, etc.)
3. Follow the Setup steps above
4. Repackage and reinstall the skills from `skills/`
5. Re-paste the project instructions
6. The wiki itself lives in your kb folder — no data is lost as long as the folder is backed up

---

## Notes

- Skills are **global** in Cowork (not project-scoped), but only make sense inside a project with the right folder structure
- Projects are **local** — not synced across machines. Recreate the project on each machine, pointing at the same folder
- For phone access, look into **Claude Code + SSH + tmux** or **Claude Code Remote Control** (`claude remote-control`)
- `raw/` should be treated as sacred — never let Claude (or yourself) edit source files
